PROTOCOLO

Lo que debe hacer el asesor antes de activar cualquier experiencia del sistema.

01

Descargar desde biblioteca privada

El asesor no recibe carpeta ni acceso directo. Descarga lo que necesita desde su sesión activa.

02

Duplicar la plantilla aprobada

Nunca se trabaja sobre el archivo maestro. Siempre sobre una copia renombrada con la propiedad activa.

03

Editar solo la zona autorizada

Propiedad, asesor, WhatsApp, ficha técnica, webhook, pixel y datos de mercado. Nada más.

04

Probar conexión antes de publicar

Enviar prueba real y confirmar llegada a Make y Airtable. Si no llega: no se publica.

05

Registrar en CRM

Toda operación activa debe estar en Airtable desde el primer día. Sin excepción.

Setup · Supabase + Vercel

CONFIGURACIÓN

Este archivo va solo en Vercel. Las plantillas viven en Supabase Storage privado.

Solo este archivo index.html va en Vercel. Las plantillas nunca se suben públicamente. Viven en el bucket privado de Supabase y solo se entregan con sesión autorizada.
Paso 01
SUPABASE

Crear proyecto → copiar Project URL y anon key → pegarlas en el bloque CONFIG de este archivo.

Paso 02
BUCKET

Crear bucket privado plantillas-neri → subir los archivos HTML con los nombres exactos del CONFIG.

Paso 03
USUARIOS

En Supabase Authentication → crear un usuario por asesor con correo y contraseña.

Paso 04
VERCEL

Arrastrar solo este index.html → publicar. El sistema ya es privado desde el primer acceso.

SQL requerido para Storage

Ejecutar en Supabase → SQL Editor:

create policy "Authenticated users can read plantillas"
on storage.objects
for select
to authenticated
using (bucket_id = 'plantillas-neri');
